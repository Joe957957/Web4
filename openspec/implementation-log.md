# OpenSpec 實作過程

## 1. 題目確認
- 選定主題為「台灣鐵路票價」
- 將主題定位為通貨膨脹觀察資料網站

## 2. 資料模型設計
- 決定欄位包含日期、商品名稱、商品價格、資料來源網站與備註
- 以 SQLite 儲存所有票價資料

## 3. 後端設計
- 使用 Express.js 建立 Web API
- 提供新增、查詢、統計摘要三種主要功能
- 首次啟動時自動建立 SQLite 資料表

## 4. 前端設計
- 使用單頁式介面呈現新增表單與查詢區塊
- 以表格方式列出所有票價變化
- 加入摘要數據卡，讓使用者快速看出最低、最高與平均價格

## 5. 文件整理
- 撰寫 OpenSpec 規格檔
- 撰寫可貼到 HackMD 或 Notion 的教學文件草稿

## 6. 交作業準備
- 專案可直接上傳 GitHub
- 若要繳交報告，建議補上本機 localhost 截圖與實際資料畫面
